# Stack — Tecnologías Aprobadas

## Stack principal (webapp-nextjs)

| Capa | Tecnología | Versión | Notas |
|------|-----------|---------|-------|
| Framework | Next.js | 14+ (App Router) | Estándar para nuevos proyectos web |
| Lenguaje | TypeScript | 5+ strict | Sin JavaScript puro en proyectos nuevos |
| Estilos | Tailwind CSS | 3+ | Utility-first, no CSS modules |
| Componentes UI | shadcn/ui | latest | Sobre Radix UI, copiable al proyecto |
| Auth + DB | Supabase | latest | PostgreSQL + Auth + Storage |
| ORM / Query | Supabase client + SQL | — | Evitar ORMs adicionales salvo necesidad |
| Estado cliente | Zustand | 4+ | Solo para estado global complejo |
| Data fetching | TanStack Query | 5+ | Para estado servidor en cliente |
| Formularios | React Hook Form + Zod | latest | Validación tipada end-to-end |
| Testing unitario | Vitest | latest | Más rápido que Jest para Vite/Next |
| Testing e2e | Playwright | latest | Estándar para acceptance tests |
| Linting | ESLint + Prettier | latest | Config base en archetype |
| Deploy | Vercel | — | Para frontend. Railway para backends |

## Configuración base requerida

Todo proyecto nuevo debe tener:

```
.env.example          ← todas las variables documentadas (nunca valores reales)
.eslintrc.json        ← config ESLint del archetype
.prettierrc           ← config Prettier del archetype  
tsconfig.json         ← strict: true
.gitignore            ← incluir .env, node_modules, .next, __pycache__
```

---

## Dependencias a evitar

```
❌ moment.js         → usar date-fns o dayjs
❌ lodash (completo) → usar lodash/es o funciones nativas
❌ class-validator   → usar Zod (en TS) o Pydantic (en Python)
❌ axios             → usar fetch nativo o ky
❌ redux             → usar Zustand o TanStack Query según caso
❌ jQuery            → no aplica en proyectos nuevos
```
